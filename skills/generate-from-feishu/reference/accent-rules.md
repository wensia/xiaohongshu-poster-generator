# 智能重点标记与总结生成规则

本文档定义了 AI 如何智能分析内容并决定：
1. 哪些词/句应该被重点标记（高亮）
2. 每页应该生成什么样的总结句

---

## 封面 vs 内容页：标记逻辑不同

| 页面类型 | 标记目标 | 标记位置 | 示例 |
|----------|----------|----------|------|
| **封面** | 核心概念词 | 标题 + 副标题 | "计划"、"否认"、"没有你" |
| **内容页** | 情感转折句 | 正文最后一句 | "好，那就不去了" |

---

## 封面标记规则

### 封面结构

```
┌─────────────────────────────┐
│  [关键词色块]                │
│                             │
│  主标题（可标记核心词）       │
│  副标题（可标记情感核心）     │
│  引用句                      │
└─────────────────────────────┘
```

### 封面标记优先级

| 优先级 | 标记位置 | 标记内容 | 示例 |
|--------|----------|----------|------|
| **1** | 主标题 | 核心动词/名词（2-4字） | "计划"、"否认"、"放弃" |
| **2** | 副标题 | 情感结果词（2-4字） | "没有你"、"不再爱" |
| **3** | 引用句 | 通常不标记（已是金句） | - |

### 封面标记分析流程

```
输入：标题 + 副标题

步骤1：分析主标题
├── 找出核心名词（计划、感情、未来）
├── 找出核心动词（否认、放弃、离开）
└── 选择情感重量更大的词（动词 > 名词）

步骤2：分析副标题
├── 找出情感结果（没有你、不再爱、心凉了）
└── 标记情感核心词

步骤3：确定标记数量
├── 主标题：1-2处
├── 副标题：0-1处
└── 总计不超过3处
```

### 封面标记示例

**示例1**：当我的计划被一个个否认

```html
<!-- 主标题 -->
<h1 class="main-title">
  当我的<span class="accent">计划</span><br>
  被一个个<span class="accent">否认</span>
</h1>

<!-- 副标题 -->
<p class="sub-title">
  直到我的计划里再也<span class="accent">没有你</span>
</p>
```

**示例2**：射手座的消费从不纠结

```html
<h1 class="main-title">
  射手座的消费<br>
  <span class="accent">从不纠结</span>
</h1>

<p class="sub-title">
  <span class="accent">值不值得</span>比什么都重要
</p>
```

### 封面禁止标记

| 类型 | 示例 | 原因 |
|------|------|------|
| 星座名称 | "射手座" | 主体标识，不是重点 |
| 时间词 | "2026"、"今年" | 背景信息 |
| 语气词 | "的"、"了"、"啊" | 无实义 |
| 整句标记 | 整个标题 | 失去重点 |

## 内容页标记规则

### 核心原则：标记情感重量最大的内容

```
❌ 错误思路：找到主题关键词 → 标记
✅ 正确思路：分析情感结构 → 找到情感核心 → 标记
```

**示例对比**：

| 内容 | 错误标记 | 正确标记 | 原因 |
|------|----------|----------|------|
| 我说我们去看海吧<br>你说太远了<br>好，那就不去了 | "去看海" | "好，那就不去了" | 后者是妥协的情感转折点 |
| 射手座花钱从不纠结<br>只看值不值得 | "花钱" | "值不值得" | 后者是核心判断标准 |
| 我以为我们会永远<br>没想到连明天都等不到 | "永远" | "连明天都等不到" | 后者是期望落空的痛点 |

---

## 情感核心优先级

按以下顺序寻找需要标记的内容：

| 优先级 | 类型 | 识别方式 | 示例 |
|--------|------|----------|------|
| **1** | 情感转折点 | 态度/立场发生180度转变 | "好，那就不去了"、"算了"、"那就这样吧"、"好吧" |
| **2** | 最终决定 | 做出的选择或决定 | "我选择离开"、"我不要了"、"就这么定了" |
| **3** | 核心矛盾 | 冲突的关键点 | "但我不能"、"可是你从来不懂"、"偏偏" |
| **4** | 情感高峰 | 情绪最强烈的表达 | "我真的很累"、"我早就放弃了"、"心凉了" |
| **5** | 反讽对比 | 表面与实际的反差 | "说好的永远"（实际已结束） |

## 信号词词库

### 转折信号词

当看到以下词语时，注意其所在句子可能是情感核心：

```
但、可是、然而、不过、只是
算了、好吧、那就、也罢
其实、说白了、到头来
却、偏偏、明明
```

### 放弃/妥协信号词

```
不要了、算了、就这样吧、随便吧
放弃、认了、接受、妥协
不想了、不争了、无所谓了
```

### 决定信号词

```
我选择、我决定、就这么办
从今以后、从此、再也不
一定要、必须、非要
```

### 情感强度词

```
真的、太、很、好
心凉、心累、失望、绝望
受够了、够了、烦了、倦了
```

## 不应该标记的内容

| 类型 | 错误示例 | 原因 |
|------|----------|------|
| **主题词** | "去看海"、"旅行"、"消费" | 只是话题，没有情感重量 |
| **时间/地点** | "去年"、"那天晚上"、"在北京" | 背景信息，非情感核心 |
| **普通动词** | "说"、"想"、"觉得" | 太泛化，无法传递情感 |
| **星座名称** | "射手座"、"双子座" | 主体标识，不是内容重点 |
| **连接词** | "因为"、"所以"、"而且" | 结构词，无实义 |

## 标记样式选择

### 三种样式及其适用场景

| 样式 | CSS类 | 适用场景 | 视觉效果 |
|------|-------|----------|----------|
| **文字变色** | `.accent` | **内容页首选**，情感核心 | 橙色加粗 |
| **下划线** | `.accent-underline` | 辅助强调、补充信息 | 橙色下划线 |
| **背景高亮** | `.accent-bg` | **仅封面或无关键词页面** | 白字橙底 |

### 色块唯一原则（重要）

**一张图片只能有一个色块！**

```
❌ 错误：.keyword 色块 + .accent-bg 色块 = 两个色块，视觉重复
✅ 正确：.keyword 色块 + .accent 文字 = 一个色块 + 文字强调
```

| 页面类型 | 有 `.keyword` 色块 | 正文强调样式 |
|----------|-------------------|--------------|
| 封面 | 有 | 用 `.accent`（文字变色） |
| 内容页 | 有 | 用 `.accent`（文字变色） |
| 无关键词页 | 无 | 可用 `.accent-bg`（背景色块） |

### 选择逻辑

```
if 页面有 .keyword 色块:
    # 内容页/封面 - 只能用文字强调
    if 是情感核心句/转折句:
        use .accent
    else:
        use .accent-underline
else:
    # 无色块页面 - 可以用背景高亮
    if 是核心转折:
        use .accent-bg
    else:
        use .accent
```

### 每页标记数量

- **最佳**：1-2 处
- **最多**：3 处
- **原则**：少即是多

```
❌ 每句都标记 → 读者不知道重点在哪
✅ 只标记1处 → 读者一眼看到核心
```

## 总结句生成规则

### 总结句的本质

总结句是对当页内容的**情感提炼**，而非内容概括。

```
正文说具体事件 → 总结说情感本质
正文说情感状态 → 总结说哲理升华
```

### 总结句类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **情感命名** | 给这段情感体验命名 | "第一次妥协"、"最后的温柔" |
| **哲理升华** | 从具体到抽象 | "放手也是一种爱"、"不是不爱是不能" |
| **自问自答** | 反思性总结 | "这就是结局吗"、"值得吗" |
| **状态描述** | 描述情感状态 | "心已经凉了"、"累了" |
| **反问留白** | 引发思考 | "又能怎样呢" |

### 生成规则

1. **字数控制**：3-8字为佳，最多12字
2. **避免重复**：不使用正文中的原话
3. **情感角度**：
   - 正文是叙事 → 总结是情感
   - 正文是情感 → 总结是哲理

### 禁止的总结方式

```
❌ 内容概括式："关于去看海的对话"
❌ 流水账式："我提议你拒绝我放弃"
❌ 直接复制："好那就不去了"
❌ 过长句子：超过12字的总结
```

## 完整分析流程

```
输入：一段文案内容

步骤1：断句
├── 按句号/换行分割成独立句子
└── 记录每句的位置（句1、句2、句3...）

步骤2：分析句间关系
├── 并列：A，B，C（无情感递进）
├── 递进：A → B → C（情感逐渐加强）
├── 转折：A，但B（B是核心）
└── 因果：因A所以B（B是结论）

步骤3：定位情感核心
├── 扫描信号词（转折/放弃/决定/情感）
├── 信号词所在句 = 候选核心
├── 如有多个候选，选最后一个（通常是结论）
└── 如无信号词，选最后一句

步骤4：选择标记样式
├── 如果核心是完整句子 → .accent-bg
├── 如果核心是短语（2-6字）→ .accent
└── 如果需要辅助强调 → .accent-underline

步骤5：生成总结
├── 提取核心情感（妥协/放弃/坚持/失望/释然...）
├── 用2-8字命名这段情感
└── 确保不与正文重复
```

## 示例分析库

### 示例1：妥协场景

```
原文：
我说我们去看海吧
你说太远了
好，那就不去了

分析：
- 句1：我说我们去看海吧 → 提议（中性）
- 句2：你说太远了 → 对方拒绝（中性）
- 句3：好，那就不去了 → 接受拒绝 = 妥协（情感核心）

信号词："好，那就" = 转折+放弃
句间关系：因果（因为你拒绝，所以我放弃）

标记决策：
- 位置：第3句完整句子
- 样式：.accent-bg（核心转折，完整句）
- 代码：<span class="accent-bg">好，那就不去了</span>

总结生成：
- 核心情感：妥协
- 情感命名："第一次妥协"
- 最终：「 第一次妥协 」
```

### 示例2：放弃场景

```
原文：
我以为我足够努力就可以
但你从来没有回应过
算了，我累了

分析：
- 句1：期望
- 句2：失望（但 = 转折信号）
- 句3：放弃（算了 = 放弃信号，累 = 情感词）

信号词：但、算了、累
句间关系：递进（期望 → 失望 → 放弃）

标记决策：
- 位置：第3句
- 样式：.accent（情感关键句，但不是完整转折）
- 代码：<span class="accent">算了，我累了</span>

总结生成：
- 核心情感：疲惫+放弃
- 哲理升华："不是不爱，是真的累了"
- 最终：「 不是不爱，是真的累了 」
```

### 示例3：坚持场景

```
原文：
所有人都说不可能
包括你
但我还是想试试

分析：
- 句1：否定
- 句2：否定加强（包括最亲近的人）
- 句3：转折坚持（但 = 转折信号）

信号词：但、还是
句间关系：转折（全世界否定 → 但我坚持）

标记决策：
- 位置：第3句
- 样式：.accent（转折句，但较短）
- 代码：<span class="accent">但我还是想试试</span>

总结生成：
- 核心情感：逆境坚持
- 情感命名："明知不可为"
- 最终：「 明知不可为 」
```

### 示例4：释然场景

```
原文：
曾经觉得离开你会死
后来发现
不过是场梦醒了

分析：
- 句1：过去的执念
- 句2：过渡
- 句3：现在的释然

信号词：后来、不过是
句间关系：对比（过去 vs 现在）

标记决策：
- 位置："不过是场梦醒了"
- 样式：.accent
- 代码：<span class="accent">不过是场梦醒了</span>

总结生成：
- 核心情感：释然
- 状态描述："醒了就好"
- 最终：「 醒了就好 」
```

### 示例5：隐忍场景

```
原文：
不是没有委屈
只是说了也没用
还不如自己消化

分析：
- 句1：承认有委屈
- 句2：说了没用（无力感）
- 句3：选择隐忍

信号词：只是、还不如
句间关系：递进（有委屈 → 没用 → 隐忍）

标记决策：
- 位置："还不如自己消化"
- 样式：.accent
- 代码：<span class="accent">还不如自己消化</span>

总结生成：
- 核心情感：隐忍
- 情感命名："习惯了一个人扛"
- 最终：「 习惯了一个人扛 」
```

## 反面示例

### 错误1：标记主题词

```
原文：射手座的消费从不纠结

❌ 错误：<span class="accent">消费</span>从不纠结
   → "消费"只是主题，没有情感重量

✅ 正确：射手座的消费<span class="accent">从不纠结</span>
   → "从不纠结"是核心态度
```

### 错误2：标记太多

```
原文：
我说我们去看海吧
你说太远了
好，那就不去了

❌ 错误：
<span class="accent">我说</span>我们<span class="accent">去看海</span>吧
<span class="accent">你说</span><span class="accent">太远了</span>
<span class="accent">好</span>，<span class="accent">那就不去了</span>
→ 标记太多，失去重点

✅ 正确：
我说我们去看海吧
你说太远了
<span class="accent-bg">好，那就不去了</span>
→ 只标记核心转折
```

### 错误3：总结重复原文

```
原文：好，那就不去了

❌ 错误总结：「 那就不去了 」→ 直接复制原文
✅ 正确总结：「 第一次妥协 」→ 情感提炼
```

### 错误4：总结太长

```
❌ 错误：「 我以为我们会一起去看海但是你拒绝了我只好放弃 」
✅ 正确：「 第一次妥协 」
```

## 检查清单

生成内容后，逐项确认：

- [ ] 标记的是情感核心，不是主题词
- [ ] 标记的是句子末尾/结论，不是开头
- [ ] 每页标记不超过 3 处
- [ ] 使用了正确的标记样式（核心用 .accent-bg）
- [ ] 总结是情感提炼，不是内容重复
- [ ] 总结字数 3-12 字
- [ ] 总结使用了「 」包裹（会自动添加）
